## 👥 About

PicoServer was born in 2024, originally an internal tool I wrote to add Web API capabilities to my own projects, and later gradually refined into a standalone library. It will maintain long-term maintenance, stable operation, with a conservative development strategy: **lightweight, AOT support**.

It is a complement to the .NET platform, providing zero-dependency, minimal-configuration Web API, WebSocket, and static file hosting capabilities for any .NET program including console, WinForm, WPF, MAUI, Avalonia, VSTO, etc. PicoServer is not a replacement for ASP.NET, it only does one thing — add Web capabilities to .NET programs, handy and out-of-the-box. Special thanks to Microsoft MVP [Alex Chow](https://docs.microsoft.com/zh-cn/previous-versions/msdn10/cc668330(v=MSDN.10)?WT.mc_id=DT-MVP-5005078) for contributing from the early days.
